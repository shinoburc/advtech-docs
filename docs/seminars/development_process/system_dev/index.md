システム開発演習
===================================

目標
-----------------------------------

- これまでの研修内容について実践的に確認を行い、研修内容の定着を目指す。
  - DB設計
  - バージョン管理
  - プログラミング
  - フレームワーク
  - プロジェクトマネジメント
  - 要件定義から実装までの流れ
    - 要件定義
    - 外部設計
    - 内部設計
- チームでの開発経験を得て、現場での開発に活かせるようにする 
  - メンバー間の情報の共有方法
  - 役割分担方法、タスク管理
  - 複数人でのコードを共有した開発
- お客様を講師がロールプレイし、顧客ありの実践的な開発経験を得て、現場での開発に活かせるようにする
  - 開発のスケジューリング
  - 顧客との交渉
- PDCA サイクルを意識した開発を行い、改善のプロセスを体験する
  - Plan（計画）→ Do（実行）→ Check（評価）→ Action（改善）
